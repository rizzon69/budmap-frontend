🔐 Feature 1 — Authentication

Authentication System

Secure multi-method login with email verification, Google OAuth, password reset, and role-based JWT tokens.

📍 Where in Code

  • Backend/routes/auth.js — all auth endpoints
  • Backend/config/passport.js — Google OAuth strategy
  • Backend/middleware/auth.js — verifyToken, role guards
  • Backend/models/EmailToken.js — verify & reset tokens in DB
  • Frontend/context/AuthContext.js — global login/logout state
  • Frontend/services/api.js → authAPI — Axios calls
  • Frontend/pages/LoginPage.js — login form UI

🔀 API Routes

POST /api/auth/login POST /api/auth/register POST /api/auth/forgot-password POST /api/auth/reset-password GET /api/auth/google GET /api/auth/me GET /api/registration/verify-email/:token POST /api/registration/resend-verification

⚙️ How It Works

  • User registers → EmailToken saved in MongoDB → verify link sent
  • Login returns JWT stored in localStorage
  • Every API request sends Authorization: Bearer TOKEN
  • middleware/auth.js decodes token, reads role from live DB
  • Reset tokens stored in MongoDB — survive server restarts ✅

💳 Feature 4 — Transactions

Transaction Management

Multi-level approval workflow for income and expenses. On approval, budget spentAmount auto-updates and email notifications are sent.

📍 Where in Code

  • Backend/routes/transactions.js — all transaction logic
  • Backend/models/Transaction.js — Mongoose schema
  • Frontend/pages/TransactionsPage.js — table + modal UI

⚙️ Key Backend Logic

📊 Feature 5 — Analytics & AI Forecasting

Analytics & AI Forecasting

Real-time dashboard, deterministic linear regression forecasting, spending patterns, and Google Gemini AI insights.

📍 Where in Code

  • Backend/routes/analytics.js — 6 endpoints
  • Backend/services/aiService.js — Gemini API calls
  • Frontend/pages/AnalyticsPage.js — charts & AI UI

🤖 AI — aiService.js

  • generateBudgetAnalysis() — sends budget + last 15 transactions to Gemini, returns JSON: health, insights, risks, recommendations
  • generateOrganizationInsights() — sends full org summary to Gemini, returns: top insights, alerts, best/worst dept
  • Uses Node.js built-in https — no extra npm package needed

🔀 API Routes

GET /api/analytics/dashboard GET /api/analytics/forecast GET /api/analytics/spending-patterns GET /api/analytics/comparison POST /api/analytics/ai-budget-analysis POST /api/analytics/ai-organization-insights

📐 Forecast Algorithm

  • Groups completed expenses into monthly buckets
  • Applies linear regression (least-squares): y = a + bx
  • Projects next N months — fully deterministic, no random values
  • Returns trend: increasing / decreasing / stable

🐛 Bug Fixes — Done During Development

5 Bugs Found & Fixed

FIX 1
Comment author name was always blank
In budgetApprovals.js: auth middleware never sets req.user.firstName/lastName. Fixed by adding getUserFullName(userId) helper that fetches real name from MongoDB on each comment.
FIX 2
Password reset tokens lost on every server restart
In auth.js: tokens were in a plain JS object (memory). Fixed by saving to MongoDB using existing EmailToken model — tokens now survive restarts, auto-expire after 1 hour via MongoDB TTL index.
FIX 3
Budget forecast gave different results every call
In analytics.js: Math.random() was used for variance. Replaced with proper deterministic linear regression (least-squares method) — same data always gives same forecast.
FIX 4
ObjectId string mismatch in analytics queries
In analytics.js: departmentId/categoryId passed as raw strings into MongoDB queries. Fixed with toObjId() helper using mongoose.Types.ObjectId.
FIX 5
Secrets exposed — no .gitignore existed
Created .gitignore at project root. Now excludes all .env files (Gmail password, Google OAuth secret, Gemini API key), node_modules, build output, and user uploads.
